# Benchmark Report

Generated by `npm run benchmark` (`src/bench/benchmark.ts`). Every number is computed
from the running `CheckoutService`, not hardcoded. Raw output is written to
`benchmarks/results/summary.json` and `benchmarks/results/duplicate_rate.csv`.

## Method

- 1,000 logical checkout intents.
- At-least-once delivery modeled with geometric redelivery (`p = 0.2`, seed `1337`),
  yielding 1,249 deliveries. The same model runs with and without idempotency.
- Without idempotency, each redelivery uses a fresh key (a system with no dedupe).
  With idempotency, redeliveries reuse the intent's stable key.
- Recovery experiment injects payment failure on one-third of intents and checks that
  compensation released every hold (no leaked inventory).

## Results

| Metric | Without idempotency | With idempotency |
|---|---|---|
| Duplicate checkout side-effect rate | 19.9% | 0.0% |
| Oversell rate (scarce SKU) | 0.0% | 0.0% |
| Failure recovery rate | — | 100.0% |
| Checkout latency p50 / p95 / p99 | — | 0.029 / 0.048 / 0.069 ms |

## Reading the numbers

- **Duplicate side effects** drop from ~20% to 0% because the idempotency record returns
  the original order for any redelivery, so no second order, hold, or payment auth occurs.
- **Oversell is 0% in both modes** because the inventory guard rejects over-reservation
  regardless of dedupe. Idempotency's value here is preventing duplicate orders and
  duplicate charges, not preventing oversell.
- **Recovery is 100%** because the saga's compensation path releases all holds on failure.

Re-run anytime with `npm run benchmark`; results are deterministic for the fixed seed.
